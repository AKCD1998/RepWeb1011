จากการเปิดให้ระบบ repo นี้ใช้เป็นเวลา 1 สัปดาห์เราพบปัญหาดังนี้
1. ถึงแม้ว่าเราจะมีการตั้งค่า ช่องกรอกข้อมูลเป็น วว/ดด/ปปปป ของ exp date ตอนรับเข้าสินค้า แต่เนื่องจาก template วันที่และเวลาของโน้ตบุ๊คแต่ละเครื่องไม่เหมือนกัน ทำให้เครื่องที่มีความแตกต่างเรื่อง format date time อย่างที่ชัดที่สุด อย่าง UK USA ที่แล้วแต่ว่าเจ้าไหนจะเอาวัน เดือน ขึ้นก่อนเวลากรอกข้อมูล ทำให้เกิดความสับสนได้ง่าย ว่าสรุปแล้วจะเอาวันที่หรือเดือนขึ้นก่อน ซึ่งใจเราอยากให้การแก้ปัญหาคือ ไม่ว่าคอมพิวเตอร์ เทมเพลตไหน UK USA หรือที่ใดในโลก เวลากรอกวันหมดอายุของเวปเราจะต้องเป็น dd/mm/yyyy เสมอง

2.เภสัชกรพาร์ทไทม์ หรือเภสัชกรอื่นๆยังไม่ชิน หรือ ยังไม่เข้าใจระบบการใช้งาน ทำให้ บางครั้งเกิดเคสที่พลาดไม่ได้เสียบบัตร ยิงสินค้าแล้วบันทึกข้อมูลลงเวปไซต์นี้ ดังนั้น เราอยากให้มี admin edit mode มีอำนาจในการจำหน่ายสินค้าออก โดยที่เป็นชื่อเรา รวมทั้งให้เป็นระบบ incident report กลายๆด้วยว่า เกิดเหตุการณ์อะไรขึ้นในครั้งนี้ ซึ่งเราในฐานะผู้ดูแลระบบขอเป็นผู้รับผิดชอบในกรณีที่ผู้ใช้งานระบบของเรายังทำให้เกิดความผิดพลาด ไว้ในอนาคตเราจะต้องหาทางแก้ไขทีหลังไม่ใช่ว่า 5 สาขา หรือ 100 สาขา เกิดความผิดพลาดก็โบ้ยเราอย่างเดียว

3. นอกจาก ระบบ incident report ที่เอาไว้จัดการเวลาทีมีการส่งมอบยาให้ผู้ป่วยโดยไม่เสียบบัตรประชาชนของผู้รับบริการใน ข้อ 2 แล้ว เราอยากได้ ระบบที่ให้ user admin สามารถ edit ข้อมูลของเลขลอตได้ รวมทั้งสามารถแก้ไขข้อมูลเลขผลิต และวันหมดอายุได้ โดยที่ ทำเป็น incident report กลายๆเหมือนกัน เพื่อให้รู้ว่ามันมีการเปลี่ยนแปลงเพราะอะไร 

ทั้งหมดเมื่อทำเสร็จสิ้นก็บันทึกความคืบหน้าของโปรเจคไว้ใน markdown ด้วยนะว่าทำอะไรไปบ้าง เพื่อที่นอกจากการอ่านข้อมูลจาก git ก็จะอ่านจาก markdown เพื่อเป็นเบาะแสว่าทำอะไรต่อ และแก้ไขอย่างไร
