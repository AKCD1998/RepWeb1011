Incident Report

เรื่อง: รายงานเหตุการณ์บันทึกเลข lot ผิด และการลบ lot ที่บันทึกซ้ำผิด สาขา 004

เรียน ผู้เกี่ยวข้อง

ขอรายงานเหตุการณ์ความคลาดเคลื่อนในการบันทึกรับสินค้าและการจัดการสต๊อกของสาขา 004 โดยมีรายละเอียดดังนี้

วันที่เกิดเหตุ: 02/06/2026
สาขา: 004
สินค้า: Neutapin 25 mg
Lot ที่ถูกต้อง: D25423
Lot ที่บันทึกผิด: ฏ25423

รายละเอียดเหตุการณ์:
ผู้ปฏิบัติงานของสาขา 004 บันทึกเลข lot ของสินค้า Neutapin 25 mg ผิดจาก "D25423" เป็น "ฏ25423" ในรายการรับสินค้า ต่อมาผู้ปฏิบัติงานได้บันทึกรายการรับสินค้าใหม่อีกครั้งด้วยเลข lot ที่ถูกต้องคือ "D25423" และมีการจ่ายสินค้าล็อตที่ถูกต้องให้แก่ผู้รับบริการแล้ว

ผลกระทบ:
จากการบันทึกเลข lot ผิด ทำให้ในระบบเกิด lot "ฏ25423" ขึ้นมาโดยไม่ตรงกับข้อเท็จจริง และทำให้สต๊อกคงเหลือของสาขา 004 มีจำนวนค้างอยู่ภายใต้ lot ที่ไม่ถูกต้อง ทั้งที่สินค้าดังกล่าวเป็น lot เดียวกับ "D25423"

แนวทางการแก้ไขที่ต้องการดำเนินการ:
1. ลบข้อมูล lot "ฏ25423" ที่เกิดจากการบันทึกผิดออกจากระบบ
2. ลบรายการ stock movement ที่เกิดจากการรับ lot "ฏ25423" ซึ่งเป็นรายการที่ไม่ควรมีอยู่
3. ปรับ stock_on_hand ของ lot "ฏ25423" ให้สอดคล้องกับการลบรายการดังกล่าว
4. คงรายการรับและรายการจ่ายของ lot "D25423" ที่บันทึกถูกต้องไว้ตามเดิม

สรุปการแก้ไขที่ต้องการ:
การดำเนินการนี้มีวัตถุประสงค์เพื่อให้ข้อมูล lot และจำนวนสต๊อกในระบบสอดคล้องกับข้อเท็จจริง โดยลบผลกระทบจากการบันทึกเลข lot ผิด และคงไว้เฉพาะข้อมูลของ lot "D25423" ซึ่งเป็น lot ที่ถูกต้อง

จึงเรียนมาเพื่อทราบและดำเนินการตามความเหมาะสม
